GitHub Guide — Hosting Your Workpad

This guide walks you through hosting this file on GitHub Pages — completely free, no technical experience needed. Once set up, your workpad will live at a URL you can open from any browser, and every time you update the file, GitHub keeps a full history so you can always go back to an earlier version.

1 Create a GitHub account (if you don't have one)

Go to github.com and sign up. Use the email swethabhagwat.myspace@gmail.com and pick a username. Your handle will be swethabhagwatmyspace.

✓ If you already have the account, skip straight to Step 2.

2 Create a new repository

  1. Once logged in, click the green "New" button (or the + icon, top-right).
  2. Name the repository exactly: tress-workpad
  3. Set it to Public (required for free GitHub Pages hosting).
  4. Tick "Add a README file".
  5. Click "Create repository".

3 Upload this file

  1. Inside your new repository, click "Add file" → "Upload files".
  2. Drag this HTML file onto the page, or click to browse and find it. Name it exactly: index.html
  3. At the bottom, in the "Commit changes" box, write a short note like: First upload of workpad.
  4. Click "Commit changes".
⚠ Make sure the file is named index.html — this is what GitHub Pages looks for.

4 Turn on GitHub Pages

  1. In your repository, click Settings (top tabs).
  2. Scroll down the left sidebar to Pages.
  3. Under "Source", select Deploy from a branch.
  4. Choose branch: main, folder: / (root).
  5. Click Save.
  6. Wait about 1–2 minutes, then refresh. A green banner will show your site URL.

Your workpad will be live at:

https://swethabhagwatmyspace.github.io/tress-workpad/

5 Updating the file when you get a new version

  1. Go to your repository on github.com.
  2. Click on index.html in the file list.
  3. Click the pencil icon ✏ (Edit this file) — top right of the file view.
  4. Select all the text and delete it (Ctrl+A, Delete).
  5. Open your new HTML file in a text editor (Notepad works), select all, copy, and paste it in.
  6. Click "Commit changes", add a note like Added chapters 4 and 5.
  7. Your site updates automatically within a minute or two.
💡 Alternatively, use the "Upload files" button to upload a replacement file — just make sure it's still named index.html.

6 Viewing your version history

Every time you commit a change, GitHub saves a snapshot. To see all past versions:

  1. Go to your repository.
  2. Click Commits (the clock icon near the top of the file list, shows something like "12 commits").
  3. You'll see a list of every save with your notes and the date.
  4. Click any entry to see exactly what changed, or click Browse files to see the full workpad as it was at that point.

7 If something goes wrong

  • Site not loading? Wait 2–3 minutes after enabling Pages. Sometimes it takes a moment to build.
  • Changes not showing? Hold Shift and click refresh in your browser (hard refresh) to clear the cache.
  • Accidentally deleted something? Go to Commits, find the last good version, click "Browse files", then copy the content of index.html and re-upload it.
  • Need help? GitHub has a free support chat at support.github.com.
✓ Because this workpad saves data to your browser's local storage, your writing is stored in the browser where you work — it's not stored on GitHub. GitHub only stores the app itself. If you clear browser data, export a backup using the PDF download first.
